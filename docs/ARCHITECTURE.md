## Overview

SAINS API is a Go web application using **Gin** as HTTP framework, **PostgreSQL** as database, **SQLC** for type-safe SQL, and **HTMX** for the admin panel frontend.

The application has two main surfaces:
1. **Public API** (`/api/v1/...`) — Authentication, subscriptions, guest access, feedback
2. **Admin Panel** (`/admin/...`) — Server-rendered HTML dashboard with HTMX interactivity

┌─────────────────────────────────────────────────────┐

│              (DI wiring, route registration)         │

             │                        │
    ┌────────▼────────┐     ┌────────▼────────┐
    │  handler/*.go   │     │  admin/*.go      │
    │  (Public API)   │     │  (Admin Panel)   │
    │  JSON responses │     │  HTML templates   │
    └────────┬────────┘     └────────┬────────┘
             │                        │
    ┌────────▼────────┐               │
    │  service/*.go   │◄──────────────┘
    │  (Business      │
    │   Logic)        │
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │  repository/    │
    │  (SQLC-generated│

    └────────┬────────┘
             │
    ┌────────▼────────┐
    │  PostgreSQL     │
    └─────────────────┘

**Data flow:** Request → Middleware → Handler → Service → Repository → DB

## Admin Handler Modularity

Each admin feature is **completely isolated** in its own Go file.
Editing one file does **NOT** affect other features.

| File | Handler Methods | Template(s) | SQL Source |
|------|----------------|------------|-----------|
| `admin_handler.go` | — (struct, render only) | `layout.html` | — |
| `admin_auth_handler.go` | `AdminAuthMiddleware()`, `LoginPage()`, `LoginPost()`, `Logout()` | `login.html` | — |
| `dashboard_handler.go` | `Dashboard()` | `dashboard.html` | multiple stats queries |
| `audience_handler.go` | `Audience()` | `audience.html` | `guest_codes.sql` → `ListAllAudience` |
| `user_handler.go` | `Users()`, `UserDetail()`, `LockUser()`, `UnlockUser()` | `users.html`, `user_detail.html` | `users.sql` |
| `guest_code_handler.go` | `GuestCodes()`, `GuestCodeDetail()`, `CreateGuestCode()`, `RevokeGuestCode()`, `DeleteGuestCode()` | `guest_codes.html`, `guest_code_detail.html` | `guest_codes.sql` |
| `subscription_handler.go` | `Subscriptions()` | `subscriptions.html` | `subscriptions.sql` |
| `pricing_handler.go` | `Pricing()`, `UpdatePriceInline()`, `TogglePlanActive()` | `pricing.html` | `pricing_plans.sql` |
| `revenue_handler.go` | `Revenue()` | `revenue.html` | `subscriptions.sql` |
| `anomaly_handler.go` | `Anomalies()` | `anomalies.html` | `anomaly_logs.sql` |
| `settings_product_audit_handler.go` | `Settings()`, `UpdateConfig()`, `Products()`, `CreateProduct()`, `UpdateProduct()`, `ToggleProductActive()`, `AuditLogs()` | `settings.html`, `products.html`, `audit.html` | `system_config.sql`, `products.sql`, `admin_audit.sql` |
| `helpers.go` | — (utilities) | — | — |

### Isolation Level

```
✅ Handler files     → FULLY ISOLATED (each file is self-contained)
✅ Templates         → FULLY ISOLATED (1 file per page, no cross-includes)
✅ SQL queries       → FULLY ISOLATED (separate .sql files per domain)
✅ Repository code   → FULLY ISOLATED (auto-generated, 1 file per query file)
⚠️ helpers.go        → SHARED (formatIDR, containsCI, resolveIPLocation, etc.)
⚠️ layout.html       → SHARED (sidebar, nav, CSS — affects ALL admin pages)
⚠️ admin_handler.go  → SHARED (struct definition, render helper)
```

## Dependency Graph

### Public API vs Admin Panel

```
Public API (handler/*.go)        Admin Panel (admin/*.go)
  └── AuthService                  └── Queries (direct)
  └── TokenService                 └── AuthService (login only)
  └── EmailService                 └── TokenService (login only)
  └── AnomalyService               └── AuditLogger
  └── MidtransService
  └── Queries

These two surfaces share NO cross-references.
Editing admin does NOT affect the Public API, and vice versa.
```

## Shared Components

### ⚠️ Files that affect multiple features

| Component | Impact | When to be careful |
|-----------|--------|-------------------|
| `helpers.go` | ALL handlers using the changed function | Changing `formatIDR()` affects ALL monetary displays |
| `layout.html` | ALL admin pages visually | Changing sidebar, nav, CSS |
| `admin_handler.go` (struct) | ALL handlers | Adding/removing fields from `AdminHandler` |
| DB migrations | Tables used by affected queries | Always check which `.sql` files query the changed table |

### Helper Function Usage Map

| Function | Used By |
|----------|---------|
| `formatIDR()` | dashboard, audience, user_detail, subscription, pricing, revenue |
| `uuidStr()` | users, guest_codes, anomalies, pricing, subscription, auth |
| `parseUUID()` | users, guest_codes, pricing |
| `containsCI()` | audience, users |
| `resolveIPLocation()` | audience only |
| `generateGuestCode()` | guest_codes only |

## What's Safe to Edit

### ✅ Safe (does NOT affect other features)

| Action | Example |
|--------|---------|
| Edit a handler file | Change `audience_handler.go` logic |
| Edit a template | Change `audience.html` layout |
| Edit a SQL query | Change `ListAllAudience` in `guest_codes.sql` |
| Add a new migration | Add `000008_xxx.up.sql` |
| Add a new handler file | Create `new_feature_handler.go` |
| Add a new template | Create `new_feature.html` |

### ⚠️ Careful (may affect multiple features)

| Action | Impact |
|--------|--------|
| Edit `helpers.go` | Check the usage map above |
| Edit `layout.html` | ALL admin pages change visually |
| Edit DB schema | Check which `.sql` queries use the changed table |
| Edit `admin_handler.go` struct | ALL handler files use this struct |
| Edit `repository/` files | ❌ NEVER — they're auto-generated by `sqlc generate` |

## Quick Reference

### "Mau Edit X, Apa Aja yang Kena?"

| Mau edit... | Files to change | Other files NOT affected |
|-------------|----------------|------------------------|
| Audience page | `audience_handler.go`, `audience.html`, `guest_codes.sql` | Everything else ✅ |
| Pricing page | `pricing_handler.go`, `pricing.html`, `pricing_plans.sql` | Everything else ✅ |
| Login flow | `admin_auth_handler.go`, `login.html` | Everything else ✅ |
| Dashboard | `dashboard_handler.go`, `dashboard.html` | Everything else ✅ |
| Users page | `user_handler.go`, `users.html` / `user_detail.html` | Everything else ✅ |
| Guest codes | `guest_code_handler.go`, `guest_codes.html` / `guest_code_detail.html` | Everything else ✅ |
| Revenue | `revenue_handler.go`, `revenue.html` | Everything else ✅ |
| Sidebar/nav | `layout.html` | ⚠️ ALL admin pages change visually |
| Currency format | `helpers.go` → `formatIDR()` | ⚠️ Dashboard, Audience, Revenue, Pricing, Subscriptions |
| Public API | `handler/*.go`, `service/*.go` | Admin panel NOT affected ✅ |

### Adding a New Admin Page

1. Create `new_feature_handler.go` in `internal/admin/`
2. Create `new_feature.html` in `internal/admin/templates/`
3. Add route in `cmd/server/main.go`
4. Add sidebar link in `layout.html`
5. Update this document's modularity table
